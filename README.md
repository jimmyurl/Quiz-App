## What is this?

Quiz Arena is a B2B SaaS platform where Tanzanian companies subscribe to assess job candidates and train employees across 12 professional categories. Unlike static quiz apps, every session hits the Claude API to generate contextually fresh questions — so candidates can't memorise answers between attempts.

Companies pay via Selcom's unified checkout (M-Pesa, Airtel Money, Tigo Pesa, and card in one flow), manage team assessments from an HR admin dashboard, and track performance on a live company leaderboard.

## Features

### Assessment engine
- **AI question generation** — calls Claude Sonnet on every session to produce unique, category-specific interview questions calibrated to the chosen difficulty
- **Graceful fallback** — if the API is unavailable, a curated bank of 500+ hand-written questions ensures the session still runs
- **12 job categories** — Software Engineering, Product Management, Finance & Accounting, HR, Marketing & Sales, Leadership, and more
- **3 difficulty levels** — Easy, Medium, Hard, each producing qualitatively different questions
- **20-second timer** per question with animated countdown that turns red under 5 seconds
- **Shuffled options** on every load — answer position changes each session
- **Streak system** — consecutive correct answers trigger a streak badge and popup
- **Per-question difficulty badge** — each question is labelled so candidates know exactly what they're facing

### Scoring & leaderboard
- Animated score ring on the result screen with correct / wrong / best-streak breakdown
- Full answer review — every question, your answer, and the correct answer
- Company leaderboard filterable by category or across all assessments
- Scores attributed to the candidate's company for internal benchmarking

### Payments — Selcom unified checkout
- Single integration covering M-Pesa, Airtel Money, Tigo Pesa, debit, and credit card
- Backend creates a Selcom order and returns a hosted payment URL opened in a new tab
- Frontend polls the backend every 3 seconds for payment confirmation
- Webhook callback activates the subscription server-side the moment Selcom confirms
- Free tier gives access to 2 categories with no payment required

## Getting started

### Run the frontend locally

No build tools needed. The frontend is a single self-contained HTML file:

```bash
git clone https://github.com/jimmyurl/Quiz-App.git
cd Quiz-App

# Windows
start index.html

# macOS / Linux
open index.html
```

Or use the VS Code Live Server extension and click **Go Live**.

> The AI question generation requires an Anthropic API key wired through the backend. Without it, the app falls back to the built-in question bank automatically — the session still runs fully, the candidate just won't see the `AI` badge on questions.

### Run the backend

```bash
cd backend
npm install
node server.js
```

The backend exposes three endpoints:

| Method | Endpoint | Purpose |
|---|---|---|
| `POST` | `/api/payment/create-order` | Create a Selcom checkout order |
| `GET` | `/api/payment/status/:order_id` | Poll payment confirmation |
| `POST` | `/api/payment/webhook` | Selcom posts the result here on completion |

### Environment variables

Create `backend/.env` — never commit this file:

```env
# Anthropic
ANTHROPIC_API_KEY=sk-ant-...

# Selcom
SELCOM_API_KEY=your_selcom_api_key
SELCOM_API_SECRET=your_selcom_secret
SELCOM_VENDOR=your_vendor_id
SELCOM_CALLBACK_URL=https://yourdomain.com/api/payment/webhook

# Server
PORT=3000
NODE_ENV=production
```

## How the AI question generation works

When a candidate clicks **Generate AI Questions**, the app sends a structured prompt to Claude Sonnet:

```
You are an expert interviewer for [category] roles in East Africa.
Generate [n] interview questions at [difficulty] level.
Return only a JSON array — no markdown, no preamble.
Each item: { question, answer, options[4], difficulty }
```

Claude returns a JSON array. The app parses it, shuffles the options, and starts the session immediately. If the API call fails for any reason — network issue, rate limit, missing key — the app silently falls back to the built-in question bank. The candidate never sees an error screen.

## Deploying to production

### Frontend — GitHub Pages

```bash
git add index.html
git commit -m "deploy: update frontend"
git push origin main
```

Enable at **Settings → Pages → Source: main branch / root**. Goes live at:
```
https://jimmyurl.github.io/Quiz-App
```

### Backend — Railway

```bash
npm install -g @railway/cli
railway login
cd backend
railway init && railway up
```

Copy the generated URL and set it as `BACKEND_URL` in `index.html` and as `SELCOM_CALLBACK_URL` in your Railway environment variables.

## HR admin dashboard

`admin-dashboard.html` is a standalone portal for HR managers with no server dependency for the UI:

- **Overview** — KPIs (employees, pass rate, active quizzes, daily completions), department performance table, weekly completions bar chart, top performers
- **Employee scores** — searchable and filterable by name, department, or pass/fail status, with one-click CSV export
- **Departments** — create departments, track per-department pass rates, drill into individual scores
- **Quiz builder** — create quizzes, write questions, assign to departments, configure pass mark, timer, shuffle, and certificate options
- **Assignments** — monitor which quiz is live in which department, completion percentages, send reminders
- **Settings** — company profile, notification preferences, billing and subscription management

## Adding question categories

Built-in questions live in the `FALLBACK_QUESTIONS` object inside `index.html`. To add a new category:

```javascript
FALLBACK_QUESTIONS.legal = {
  easy: [
    {
      question: "What is the difference between civil and criminal law?",
      answer: "Civil law resolves disputes between individuals; criminal law involves the state prosecuting offences",
      options: [
        "They are the same",
        "Civil law resolves disputes between individuals; criminal law involves the state prosecuting offences",
        "Criminal law only applies to violent crimes",
        "Civil law is enforced by the police"
      ]
    }
  ],
  medium: [ /* ... */ ],
  hard:   [ /* ... */ ]
};
```

Then add a card to the `#category-grid` in the HTML. The Claude prompt automatically adapts to any category string you pass — no prompt engineering needed per category.
